# 服务器被 OpenClaw 搞挂了

> 日期：2026-03-10

最近这段时间 OpenClaw 龙虾的热度属实太高了，睁眼闭眼全是龙虾。记得前段时间有  ruanyf 发了一篇帖子说有网站统计了一下全球暴露在公网上的龙虾 IP 地址。



<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260310163222175.png" alt="image-20260310163222175" style="zoom:50%;" />

地址在这里 [openclaw.allegro.earth](https://t.co/gWZ92rkXAR)。



当时我还不以为意，然后今天，我才发现了事情的不对劲。



我在登录云服务器的时候，总是出现 ssh 被直接 kill 掉的情况。



![image-20260310163640802](https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260310163640802.png)



因为我 ssh 登录服务器总是断连，所以我在 mac 本地用 claude 排查了一下问题的原因。



<img src="https://cdn.jsdelivr.net/gh/doggaifan/picbed/682ac91a453381d48d14635876e97e6d.png" alt="682ac91a453381d48d14635876e97e6d" style="zoom:50%;" />



他说从外部来看是没问题的。



之后我让 OpenClaw 排查了一下服务器断连的原因，他排查是有人正在暴力使用一些简单且重复的密码（root、test、admin）对我的 ecs 进行攻击。



然后 OpenClaw 采取了一种策略，只要有IP连续 3 次登录失败，就会被自动封禁24小时，那些暴力破解的机器人一上来就会被挡住。我觉得这样可以，因为总不会有二笔能输错三次密码吧。



然后。。。。。。



![image-20260310214729612](https://cdn.jsdelivr.net/gh/doggaifan/picbed/image-20260310214729612.png)



求求了，长点心吧。
